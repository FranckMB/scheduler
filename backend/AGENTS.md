## Zone gotchas (facts not in the root docs)

1. **All commands run in the php-fpm container** — `backend/Makefile` wraps `docker compose exec`. `composer`/`php bin/console` on the host fail.
2. **CS-Fixer rule set** is `@Symfony` + `@Symfony:risky` + `@PHP84Migration` + `@PHP80Migration:risky` (see `.php-cs-fixer.dist.php` — the canonical source).
3. **Entities use string UUIDs** (no auto-increment) and scalar FK columns (`clubId`, `seasonId`… as strings, no object associations).
4. **`declare(strict_types=1)`** in every file; single-action controllers use `#[AsController]` + `__invoke()`.
5. **Redis lock** `ClubGenerationLock` (SETEX NX + release token) serialises generation per club.
6. **Cache pool `cache.schedule`** — dedicated Redis pool for engine payloads, TTL 4h, invalidated by `CacheInvalidationListener`.
7. **Slot lock levels** = `NONE` / `HARD` only. `SOFT` is **rejected** at the manual-edit lock endpoint (ENG-21): the engine never consumed a soft-lock penalty, so a SOFT lock had zero placement effect — no such no-op weight is built or sent.
8. **Test DB required** before PHPUnit: `make db-init-test` once, then `make phpunit` (`--group phase1`).
9. **Stuck-schedule watchdog** `app:schedules:reconcile-stuck` (BCK-01) must run on a **cron in prod** (e.g. every 10 min): it fails `GENERATING` schedules abandoned by a crashed/OOM worker. Terminal-status nets are detailed in root `docs/project-map.md` §2.4.
9b. **Period-reminder cron** `app:periods:remind` (cockpit palier C) runs **daily in prod** (e.g. @8h): emails a club's managers (owner+admin) when a period starts within 14 days and still has NO overlay plan. It scans a **[today, today+14] window** and picks the milestone **bucket** (14/7/3 days) — a window (not an exact date) so a **missed run catches up**. Idempotence via `period_reminder_log` (one row per `(calendar_entry_id, threshold)`, global table, no RLS) — never re-sends a milestone. Exits **non-zero if any email fails** (cron monitors see mailer outages). Never auto-acts (mirrors the in-app radar). Needs `FRONTEND_BASE_URL` (empty → omits the link). `--dry-run` / `--date=YYYY-MM-DD` (strict, rejects rollovers) for rehearsal. Club-walk + per-club GUC like the BCK-01 watchdog; managers via `ClubUserRepository::findManagementEmails` (raw DBAL, empty-GUC safe). Mailer is **sync**.
10. **Dated constraints excluded from generation** — a `Constraint` with `calendarEntryId` set belongs to a `CalendarEntry` period (cockpit, palier A) and MUST NOT feed base-plan generation. Generation/validation/report load constraints via `ConstraintRepository::findPermanentByClubSeason` (`calendarEntryId IS NULL`), never `findByClubSeason`. See `specs/courantes/accueil-cockpit-temporel.md` §9ter.
11. **Sticky cockpit unlock** — `Season.socleValidatedAt` is stamped once when the season's baseline schedule is first `VALIDATED` (or a `VALIDATED` schedule is set as baseline); **never reset** (reopen keeps it). Exposed on `/api/me`. Gates the frontend cockpit vs work-loop.
12. **School holidays = global reference** — `school_holiday_period` has **no club_id / no RLS** (public national data, like `sport`). Two feeders, both idempotent by `(zone, holidayType, schoolYear)`: **(a)** `app:school-holidays:import` — pulls the official Éducation nationale ODS API (`data.education.gouv.fr`, dataset `fr-en-calendrier-scolaire`), current season + N+1, for **all 13 zones** (A/B/C, CORSE, DOM/TOM); **(b)** `app:school-holidays:seed` — offline JSON fallback (`data/school-holidays.fr-metropole.json`), **no network**. Import rules (`FrenchSchoolCalendarMapper`): filter `population="-"` (term vacations) or `"Élèves"` (summer, drops the `Enseignants` split); a record is a vacation iff **> 3 business days (Mon–Fri) AND label ≠ `Pont …`** (excludes ponts/short breaks/1-day markers); `holidayType` is an **open slug** derived from the label (métropole + overseas differ: `carnaval`, `ete_austral`, `apres_1ere_periode`…). Read via `GET /api/school-holidays` (filtered by the club's `schoolZone` + a window). `Club.schoolZone` is derived from the FFBB code by `SchoolZoneResolver` at register (`app:clubs:backfill-school-zone` for existing clubs, dry-run by default). FFBB code = **3-letter league + 4-digit zero-padded department + sequence**; department → one of the **13 codes** in `SchoolZoneResolver::ZONES` (e.g. `GES0067060` → 67 → `B`; `COR0020001`/`2A`/`2B` → `CORSE`; `GUY0973021` → 973 → `GUYANE`; 971–988 → territory). Unknown shapes / territories not in the map → null → manually editable `schoolZone`. **A second global reference `public_holiday`** (same no-club_id / no-RLS shape) holds jours fériés, fed by `app:public-holidays:import` from the etalab API (`calendrier.api.gouv.fr/jours-feries/{zone}.json`, 10 files, current calendar year + N+1): `zone='NATIONAL'` for métropole fériés (all clubs) + territory extras = **diff territoire − métropole** tagged with the DOM/TOM zone codes (`PublicHolidayMapper`, e.g. Abolition de l'esclavage → GUADELOUPE). `NATIONAL` is a synthetic marker, **not** in `ZONES`. Column is `holiday_date` (`date` is reserved). Read via `GET /api/public-holidays` (NATIONAL ∪ `Club.schoolZone`; null zone still returns NATIONAL). **Display-only — never feeds the solver.** Alsace-Moselle + Saint-Barthélemy/Saint-Martin intentionally not imported.
13. **Cockpit conflicts** — `GET /api/calendar-entries/{id}/conflicts`: for a period entry, reads its closed venues from dated FACILITY constraints, then the **baseline** schedule's slot templates on those venues whose ISO weekday (`ScheduleSlotTemplate.dayOfWeek` = **1=Mon..7=Sun**) falls in the window → returns concrete conflicting dates. Palier A **surfaces** conflicts, moves nothing (overlay = palier B).
14. **Period overlays (palier B)** — a `Schedule` with `calendarEntryId` set is the OVERLAY of a period (inverse of `CalendarEntry.overlayScheduleId`, stamped server-side at POST). Generation branches on it: `GenerateScheduleHandler` calls `ScheduleConstraintBuilder::buildForOverlay` (closure = permanent + dated constraints; holiday = dated only + full structure), which **BYPASSES the schedule-input cache** and **skips `assignBaselineIfFirst`** (an overlay never becomes the season baseline). Deletion of an overlay/period goes through `OverlayManager` (slots + diagnostics have no FK cascade — purge them). ⚠ **`config.type=venue_closed` (the dated constraint the cockpit creates) is a NO-OP in the engine** — `buildForOverlay` expands each closed venue into per-team FACILITY HARD `config.forbiddenVenueId` constraints, which the engine honors (`constraints.py` → `forbidden_assignments`). If the dated constraint ever needs to be consumed as-is → engine change → re-validate with the user.
15. **Destructive baseline reopen** — reopening the season BASELINE while overlays exist → **409 `{code:"overlays_exist", count, overlays[]}`**; the client re-POSTs `{"confirmDeleteOverlays": true}` → `OverlayManager` deletes the overlays (entries + dated constraints KEPT — the period falls back to "signalée"). Guarded by `ReopenScheduleTest`.
16. **Interlinked-entity cascade delete** — team/venue/coach/availability-slot rows carry **no ORM association or DB FK** (all links are plain guid columns), so a bare `remove()` orphans reservations, coach links, constraints and the materialised HARD `ScheduleSlotTemplate` the solver reads back — an **orphan reservation still forces a phantom HARD pre-placement**. `EntityCascadeDeleter` (called from each `*StateProcessor` via the `AbstractStateProcessor::cascadeBeforeDelete` hook) purges the deleted row's logical children — **team** → reservations · team_coach · coach_player · slot_templates · scoped constraints · tag assignments · **fixtures · competitions · schedule_diagnostics** ; **venue** → slots · reservations · templates · diagnostics · FACILITY constraints · null-out `forcedVenueId`/`fixture.venueId` ; **coach** → team_coach · coach_player · diagnostics · COACH constraints · null-out `template.coachId` ; **slot** → its reservations + only their **HARD** templates (SOFT/NONE solver results on the same venue/day/time are RESULTS, not pins — never touched). Scoped by `clubId + seasonId` **and** RLS (never crosses a club). Filters disabled via the `DisablesTenantFilters` trait (shared with `SeasonDataPurger`/`PurgeOrphansCommand`) — the reserved-word `constraint` table breaks their alias; NOT re-enabled (Doctrine drops a disabled filter's bound params, and the DELETE 204 needs no further scoped read). `app:purge-orphans` (`--force`) one-shot-cleans orphans left by pre-cascade deletes (orphan reservations, dangling links, HARD templates whose slot/team is gone, constraints whose scopeTargetId is gone). ⚠ **Known gap**: a constraint that references a deleted entity only inside its `config` JSON (multi-target family) or via `parentConstraintId` is NOT cleaned — scope+scopeTargetId only. Guarded by `CascadeDeleteApiTest` (§7.1: cascade completeness + tenant isolation with a shared-teamId cross-club case).
